## Getting Started[​](#getting-started "Direct link to Getting Started")

1. **Fork the repository**
2. **Clone your fork**:
   ```
   git clone https://github.com/warpy-ai/script.gitcd script
   ```
3. **Create a branch**:
   ```
   git checkout -b feature/your-feature-name
   ```

## Building[​](#building "Direct link to Building")

See the [Getting Started](/getting-started) guide for build instructions.

## Testing[​](#testing "Direct link to Testing")

Run the test suite:

```
# Run all testscargo test --release# Run specific IR testscargo test --release ir::# Output: 94 tests passed
```

## Code Style[​](#code-style "Direct link to Code Style")

* Follow Rust conventions
* Use `rustfmt` for formatting
* Write tests for new features
* Update documentation

## Project Structure[​](#project-structure "Direct link to Project Structure")

```
script/├── src/│   ├── main.rs              # Entry point│   ├── compiler/            # Rust compiler (SWC-based)│   ├── ir/                  # SSA IR system│   ├── runtime/             # Native runtime kernel│   ├── vm/                  # Stack-based VM (debug)│   ├── backend/             # Native backends (Cranelift, LLVM)│   └── stdlib/              # Standard library├── bootstrap/               # Self-hosting compiler├── std/                     # Standard prelude└── examples/                # Example programs
```

## Areas for Contribution[​](#areas-for-contribution "Direct link to Areas for Contribution")

* **Language features**: ES modules, async/await, more stdlib
* **Performance**: Optimizations, profiling, benchmarks
* **Documentation**: Examples, guides, API docs
* **Tooling**: Formatter, linter, LSP
* **Testing**: More test cases, edge cases

## Submitting Changes[​](#submitting-changes "Direct link to Submitting Changes")

1. **Write tests** for your changes
2. **Update documentation** if needed
3. **Run the test suite** to ensure everything passes
4. **Submit a pull request** with a clear description
