# Rolls - Official System Libraries

**Rolls** are the official system libraries for Oite, analogous to Rust's standard library crates. Each Roll provides specific functionality that builds on Oite core's primitives.

## What's in This Section

- **[Design](/rolls/design)** - Architecture and library catalog
- **[Memory Model](/rolls/memory-model)** - Runtime memory model and ownership

## Overview

Rolls provide:

- **Core System Libraries** - Async runtime, TLS, HTTP, WebSocket
- **Standard Library** - File system, JSON, math, date, strings, arrays
- **Native Performance** - All libraries compile to native code
- **Optional Dependencies** - Only include what you need

## Architecture

```
User App Code
     │
     ▼
┌────────────────────────────────┐
│  Rolls (official system libs)  │  <-- THIS SECTION
│  @rolls/http, @rolls/tls, etc  │
└────────────────────────────────┘
     │
     ▼
Oite Core (compiler, ABI, basic async)
```

## Available Rolls

### Core System Rolls

| Roll | Purpose |
|------|---------|
| `@rolls/async` | Work-stealing executor, io_uring |
| `@rolls/tls` | TLS encryption via rustls |
| `@rolls/http` | HTTP/1.1, HTTP/2 server |
| `@rolls/websocket` | WebSocket protocol |

### Standard Library Rolls

| Roll | Purpose |
|------|---------|
| `@rolls/fs` | File system operations |
| `@rolls/path` | Path utilities |
| `@rolls/json` | JSON parse/stringify |
| `@rolls/math` | Math functions |
| `@rolls/date` | Date/time handling |
| `@rolls/crypto` | Cryptographic operations |

## Usage Example

```javascript
// Import from Rolls
import { HttpServer, HttpRequest, HttpResponse } from "@rolls/http";
import { TlsConfig } from "@rolls/tls";

// Create HTTPS server
let server = new HttpServer({
    port: 443,
    tls: TlsConfig.fromFiles("cert.pem", "key.pem")
});

server.on("request", (req: HttpRequest): HttpResponse => {
    return new HttpResponse(200, { "Content-Type": "text/plain" }, "Hello!");
});

await server.listen();
```
